# Kelly Allocation Lab Design Contract

## Audience and question

개인 퀀트 연구자가 선택한 기간 또는 직접 입력한 가정에서 성장률·위험·Kelly 비중의 관계를 검증한다. 첫 화면은 “어떤 입력으로 어떤 비중과 성장률이 계산되었는가”를 먼저 답한다.

## Information hierarchy

1. 모드, 자산, 공식 분석기간, 금리·비용
2. Full Kelly와 선택 비중, 1배·2배 기대 로그성장
3. 성과지표와 기간 탐색 스냅샷
4. 누적자산, 낙폭, 성장률-레버리지
5. 다자산 비중·상관행렬·리밸런싱 상세
6. 계산 정의, 데이터 출처, 제한사항

## Interaction contract

- 날짜 입력은 공식 분석기간이다.
- 차트의 양끝 브러시는 탐색기간만 바꾼다.
- `분석기간에 적용`을 눌러야 공식 지표·Kelly·표가 바뀐다.
- 잘못된 범위나 데이터 실패는 기존 공식 결과를 보존한다.
- 외부 데이터가 없으면 직접 가정 모드는 계속 작동하고 역사 모드는 명시적으로 unavailable을 보인다.

## Visual contract

- 단일 계열은 청색 한 뿌리와 중립색을 사용하고 불필요한 범례를 숨긴다.
- 수익률, 낙폭, 성장률-레버리지, 비중, 상관, 리밸런싱을 서로 다른 차트로 분리한다.
- 색상만으로 상태를 구분하지 않고 선 종류·마커·직접 라벨을 병행한다.
- 차트 제목은 중립적으로 유지하고 기간·단위·기준은 부제에 둔다.
- 모바일에서는 페이지 전체가 가로 스크롤하지 않으며 넓은 표와 차트만 내부 스크롤한다.

## Source and status contract

정량 카드·차트·표는 자산 계약의 source, returnBasis, dataAsOf를 노출한다. 허용 상태는 `published`, `live_api`, `stale`, `degraded`, `unavailable`, `ruin`뿐이다.
